# TestMail Hub 计划书实现对照表

更新日期：2026-04-01

这份文档用于把 [产品与研发计划书](./产品与研发计划书.md) 里的阶段目标，映射到当前仓库的真实实现状态，方便发版、对外说明和后续排期。

## 状态定义

- `已完成`：阶段核心目标和主要验收点已经落地
- `大部分完成`：主干能力已完成，但仍有少量缺口
- `部分完成`：已经有基础能力，但离计划目标还有明显差距
- `未开始`：当前仓库里还没有对应闭环

4. `Phase 7 生命周期与策略` 仅在现网确有需要时再补高价值项，其余高级观测和 SLA 后置

## 当前精简推进原则

- 只继续做会影响现网可用性、权限安全、审计闭环和 Cloudflare 多域名主线的功能
- 不为“看起来完整”补演示层、包装层和低频后台花活
- 新功能默认优先做闭环和稳定性，不追求额外抽象层与概念完备

## 明确后置项

- `Phase 8` 整体后置：Cypress 示例、销售首页、价格页、Demo 数据、演示账号当前都不做
- `Phase 2` 的 `workspace` 独立抽象后置，当前继续沿用现有项目 / 环境 / 邮箱池三层模型
- `Phase 4` 的更统一事件命名、外部告警路由增强后置，除非现网接入已经明显受阻
- `Phase 5` 的更多 provider 落地后置，当前只围绕 Cloudflare 主链路继续补
- `Phase 7` 的更细定时任务告警、SLA、批量运维动作后置，除非现网已经出现治理压力
- `Phase 1` 的提取质量指标面板、规则命中解释性增强后置，除非提取准确率已明显影响使用
