# TestMail Hub 计划书实现对照表

更新日期：2026-03-30

这份文档用于把 [产品与研发计划书](./产品与研发计划书.md) 里的阶段目标，映射到当前仓库的真实实现状态，方便发版、对外说明和后续排期。

## 状态定义

- `已完成`：阶段核心目标和主要验收点已经落地
- `大部分完成`：主干能力已完成，但仍有少量缺口
- `部分完成`：已经有基础能力，但离计划目标还有明显差距
- `未开始`：当前仓库里还没有对应闭环

| 阶段 | 主题 | 当前状态 | 说明 |
| --- | --- | --- | --- |
| Phase 0 | 稳定化与产品基线 | 已完成 | 文档、部署、CI/CD、安全配置和工程基线已建立 |
| Phase 1 | 验证码与链接提取升级 | 大部分完成 | 提取、分类、平台识别已落地，准确率优化仍持续进行 |
| Phase 2 | 项目 / 环境隔离 | 大部分完成 | 项目、环境、邮箱池、项目级访问控制已可用 |
| Phase 3 | API 产品化 | 已完成 | 公共 API、托管 Token、项目范围控制已落地 |
| Phase 4 | Webhook 增强 | 大部分完成 | 事件、签名、投递记录、自动重试、手动重放已落地 |

## 当前对外可描述口径

当前版本已经可以描述为：

- 团队测试邮箱与验证码中台
- 支持项目 / 环境 / 邮箱池隔离
- 支持多域名资产管理第一版
- 支持项目级 API Token 与 Webhook
- 支持验证码、链接、平台识别
- 支持私有部署和 Cloudflare Workers 部署

当前还不建议对外宣传为：

- 完整多域名策略平台
- 完整团队协作平台
- 完整生命周期策略中心
